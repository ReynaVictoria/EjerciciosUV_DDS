Un objeto es una colección de propiedades.
Una propiedad es una asociación entre un nombre o clave, y un valor. Su valor puede ser una variable o una función.
Si una propiedad tiene por valor una función, se le llama método.

Es un lenguaje prototipado. Los objetos heredan de otros objetos y en última instancia dle prototipo de Object


Notación de punto:
objeto.propiedad

Notación de corchetes:
objeto["propiedad"]

Una vez definido un objeto se pueden crear nuevas propiedades

La cadena de prototipos
Todo objeto heda de Object

1. juan es una instancia de Persona
Tiene sus propias propiedades (como nombre)
Su prototipo interno ([[Prototype]]) apunta a Persona.prototype
2. Persona.prototype
Es un objeto que contiene métodos compartidos (como saludar)
Es compartido por todas las instancias creadas con new Persona()
3. Persona.prototype.__proto__
Apunta a Object.prototype, el prototipo base de todos los objetos
4. Object.prototype.__proto__
Es null, porque está en la cima de la cadena de herencia

juan instanceof Persona; // true
juan instanceof Object; // true
Object.getPrototypeOf(juan) === Persona.prototype; // true
Object.getPrototypeOf(Persona.prototype) === Object.prototype; // true

No se puede usar arrow function dentro de una función constructora
Se puede omitir {} si es de una sola linea

Que pasa con this en las arrow functions?
No tienen su propio this, sino que captural su valor del contexto en el que fueron definidas. Lo heredan.

Arrays o vectores
Son conjuntos de datos ordenados por posiciones asociados en una variable

Constructor Array -> espacios vacios, NO undefined

Quitar el primer elemento
